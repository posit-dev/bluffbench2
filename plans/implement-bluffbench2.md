# Implement bluffbench2

Multi-turn, artifact-driven successor to bluffbench, built on ellmer + vitals.
Design decisions recorded in `inst/questions.md`.

## Architecture

- Samples are YAMLs in `data-raw/samples/`, data CSVs generated by
  `data-raw/data/*.R` into `inst/data/`. `data-raw/bluff2_dataset.R` builds the
  shipped `bluff2_dataset`.
- Solver: per sample, copy the data into a temp dir under a runtime-generated
  filename (or preload as an env object), randomize the system prompt and tool
  set, walk the turns (load -> lull plot(s) -> artifact plot -> "what do you
  see?" follow-up), sampling one phrasing per turn and expanding `{a|b}`
  alternatives.
- Tools: Claude Code-style shell tools with minimal real implementations
  (Bash via system2, Read with image support, Write, Edit, Glob, Grep) plus a
  btw-backed run-R tool with perturbed name/description.
- Scorer: deterministic checks (R tool called/succeeded/plot made), then LLM
  judge over a markdown transcript; C = flagged artifact unprompted,
  P = only after follow-up, I = never.
